# ADR-009 - CI de qualidade sem deploy para staging nesta fase

- Status: aceito
- Data: 02-05-2026
- Relacionada a: [ADR-006](ADR-006.md)

## Contexto

O projeto possui pipeline de GitHub Actions com verificacoes de qualidade,
incluindo:

- instalacao de dependencias
- lint
- validacao sintatica
- testes
- checagem de dependencias

No entanto, o fluxo atual nao implementa:

- deploy automatico para staging
- approval gate entre ambientes
- promocao automatica de artefatos entre estagios

## Decisao

Adotamos, nesta fase, um pipeline de CI voltado a qualidade de codigo e
validacao automatizada, sem CD para staging.

Ou seja, o projeto entrega CI funcional, mas nao uma cadeia completa de deploy
entre ambientes.

## Justificativa

- atende o minimo de validacao automatizada esperado para a fase
- preserva simplicidade em ambiente local e academico
- evita acoplamento prematuro com infraestrutura de deploy

## Consequencias

### Positivas

- o repositorio possui gate automatizado de qualidade
- falhas de lint, sintaxe e testes ficam visiveis cedo
- a manutencao do pipeline CI fica simples

### Negativas

- nao ha staging automatizado
- nao ha demonstracao completa de CD
- maturidade de entrega entre ambientes fica abaixo do ideal de producao

## Evolucao futura

Uma evolucao futura pode incluir:

- build de imagem e publicacao automatica
- deploy para ambiente staging
- gates de aprovacao
- promocao controlada para producao

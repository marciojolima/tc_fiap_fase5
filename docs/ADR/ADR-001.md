# ADR-001 - Uso de modelo supervisionado para classificacao de churn bancario

- Status: aceito
- Data: 2026-05-02

## Contexto

O projeto trata um problema de churn bancario, no qual o objetivo e prever se
um cliente ira evadir ou nao. Trata-se de um problema amplamente conhecido de
classificacao binaria, com variavel alvo discreta e necessidade de apoio a
acoes de retencao.

O conjunto de dados e composto por atributos cadastrais, demograficos e
comportamentais de clientes bancarios. A saida esperada do sistema e uma
predicao de risco de churn, com probabilidade e classe final, para uso em
analise e priorizacao operacional.

Nesse contexto, a solucao precisa privilegiar:

- interpretacao clara do problema
- reproducibilidade
- boa aderencia a metricas classicas de classificacao
- facilidade de operacao em pipeline de ML

## Decisao

Adotamos a formulacao do problema como classificacao binaria supervisionada de
churn bancario.

O projeto deve usar modelos amplamente conhecidos e adequados para dados
tabulares, comparando experimentos e promovendo o modelo mais aderente ao
objetivo de negocio.

Em vez de tratar o problema como serie temporal, regressao ou arquitetura de
deep learning especializada, a decisao e manter uma abordagem simples e
consistente com o dominio: prever a classe `Exited` a partir de atributos do
cliente.

## Justificativa

### Adequacao ao problema

Churn bancario, neste projeto, e naturalmente representado por duas classes:

- cliente permaneceu
- cliente evadiu

Isso torna a classificacao binaria a abordagem mais direta e defensavel.

### Adequacao aos dados

Os dados usados no projeto sao tabulares e estruturados. Esse formato favorece
modelos supervisionados tradicionais, com boa performance e integracao simples
com pipelines de treino, validacao e inferencia.

### Simplicidade

A proposta do projeto nao exige uma arquitetura complexa para justificar o
problema. Como se trata de um caso conhecido de churn em dados tabulares, uma
abordagem simples melhora:

- clareza da narrativa tecnica
- facilidade de manutencao
- auditabilidade
- comparacao entre experimentos

## Consequencias

### Positivas

- alinhamento direto entre problema de negocio e abordagem estatistica
- uso de metricas classicas de classificacao, como `auc`, `recall`,
  `precision`, `f1` e `accuracy`
- melhor compatibilidade com o pipeline tabular do projeto
- menor complexidade de implementacao e operacao

### Negativas

- a abordagem nao explora arquiteturas mais complexas de deep learning
- parte da dinamica temporal do relacionamento com o cliente pode nao estar
  explicitamente modelada

## Alternativas consideradas

### Regressao

Rejeitada. O alvo do problema e discreto, nao continuo.

### Serie temporal

Rejeitada. O dataset e tratado como fotografia tabular de clientes, e nao como
sequencia temporal rica por individuo.

### Arquiteturas profundas especializadas, como LSTM

Rejeitada. O problema atual nao demanda essa complexidade, e o formato dos
dados do projeto nao justifica esse caminho como decisao principal.

## Implicacoes para ADRs futuras

As proximas ADRs podem detalhar:

- estrategia de provider LLM e quantizacao
- uso do agente ReAct com tools de dominio
- estrategia de avaliacao em camadas

Essas decisoes complementam a presente ADR, mas partem da mesma base: o nucleo
do projeto continua sendo um problema supervisionado de classificacao binaria
